# Network Request Capture

## Goal

Capture fetch/XHR network requests and responses as factual Observation IR events during Browser Worker recording.

This capability extends browser lifecycle recording. It does not infer action-request links, generate Evidence Graph edges, or build Workflow IR.

## Event Types

### `network.request`

Payload fields:

- `requestId`: stable request ID generated by Browser Worker.
- `method`: HTTP method.
- `url`: request URL.
- `resourceType`: Playwright resource type.
- `headersRef`: artifact URI for request headers.
- `bodyRef`: artifact URI for request body when a request body exists.
- `timing`: Playwright request timing object.

Artifact refs:

- `artifact://{sessionId}/{requestId}/request.headers.json`
- `artifact://{sessionId}/{requestId}/request.body` when a body exists.

### `network.response`

Payload fields:

- `requestId`: same ID as the matching request event.
- `method`: HTTP method from the request.
- `url`: response URL.
- `resourceType`: Playwright resource type.
- `status`: HTTP response status.
- `statusText`: HTTP response status text.
- `contentType`: response content type when available.
- `headersRef`: artifact URI for response headers.
- `bodyRef`: artifact URI for response body.
- `timing`: Playwright request timing object.

Artifact refs:

- `artifact://{sessionId}/{requestId}/response.headers.json`
- `artifact://{sessionId}/{requestId}/response.body`

## Filtering

P0 keeps only fetch/XHR style observations and filters obvious static assets:

- Resource types: `document`, `font`, `image`, `media`, `script`, `stylesheet`.
- URL extensions: `.avif`, `.css`, `.gif`, `.ico`, `.jpeg`, `.jpg`, `.js`, `.map`, `.png`, `.svg`, `.webp`, `.woff`, `.woff2`.

## Tags

Browser Worker assigns basic tags:

- `json`: request accept/content-type or response content-type includes `application/json`.
- `graphql`: URL path includes `graphql` or JSON body includes `query` / `operationName`.
- `sensitive`: headers include `authorization`, `cookie`, or `set-cookie`.

## Acceptance Criteria

- Browser Worker records at least one fetch/XHR request.
- Backend API stores matching `network.request` and `network.response` events.
- Request and response events share the same `requestId`.
- Events include method, URL, response status, tags, timing, and artifact reference conventions.
- Typecheck, lint, tests, build, and smoke verification pass.

## Non-scope

- Persisting artifact bytes.
- Header/body redaction.
- Action-request linking.
- Evidence Graph generation.
- Workflow IR generation.
