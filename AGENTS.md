## Documentation Rules

- Use GitHub-compatible relative Markdown links.
- Do not use Obsidian wiki links in committed docs.
- Internal engineering docs go under `docs/internal/`.
- Public Chinese docs go under `docs/zh/`.
- Public English docs go under `docs/en/`.
- Capability specs go under `docs/internal/specs/`.
- Implementation plans go under `docs/internal/plans/`.
- Major decisions go under `docs/internal/adr/`.
- Module-level maintenance notes go under `docs/internal/modules/`.
- Test strategy, smoke tests, and verification notes go under `docs/internal/testing/`.
- Operational/debugging instructions go under `docs/internal/runbooks/`.
- Do not commit real cookies, tokens, private request bodies, captured credentials, or sensitive screenshots.
